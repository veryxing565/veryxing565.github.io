hypiper个人使用说明

Created2025-05-05|Updated2025-05-13
|Post Views:

在进行一个类群的系统发育分析过程中，在利用Othofinder或是其他方法基于基因组或转录组数据得到一组单拷贝基因后，为了更快捷的获取其他所需物种的该组单拷贝基因，往往会考虑从基因组或转录组测序数据入手，因此会使用aTRAM或Hybpiper。在尝试使用aTRAM时没有成功，Hybpiper倒是跑通了。因此，在此只简要介绍个人使用Hypbpiper的记录，并且针对转录组数据。以前也有看到使用aTRAM从基因组低深度测序数据中，组装对应的单拷贝基因，再以后尝试吧。

0 软件安装

sratoolkit安装

hybpiper安装

1 数据选择

转录组的数据主要源自NCBI，通过搜索一个分类群，然后挑选物种及其转录组数据。

若一个物种的RNA-seq数据数量较多，则优先选择该物种，并且尽可能下载该物种的多个RNA-seq数据，主要利用sratoolkit。如在昆虫纲的Grylloblattodea中，而Grylloblattella pravdini的转录组有9个，数量最多，因此倾向于选择该物种，下载其尽可能多的转录组数据。而本文假设下载SRR11533876、SRR11533877 和SRR11533888三个（实际上认为应该更多一些）。

使用fastp进行质控

3 Hybpiper运行

Hybpiper的运行需要近缘物种的确定的单拷贝基因序列，我们采用蛋白质序列，并且测序数据的格式为fastq，因此，比对软件使用blast。而Hypiper的运行可以对单个转录组分别进行，也可以将多个转录组数据合并在一起，再使用Hybpiper组装单拷贝基因。在实际测试中，合并后的转录组组装效果更好一些。

3.1 单个转录组分别组装

单个转录组的单拷贝基因组装，使用blast。然后通过python汇集多个转录组的组装结果，并剔除序列长度和参考序列差异过大的，以及N占比过高的序列，若还有多个序列候选，则选择与参考序列最相似的序列，相似性的分析需以来mafft。

3.2 转录组先合并再组装

即先将多个转录组合并，在利用hybpiper组装单拷贝基因。

3.3 利用diamond加速组装

利用diamond完成在hybpiper组装中的比对工作，速度比默认的blast快很多。经过实测，组装得到的基因数量会略微少于blast，时间不够的话，可以考虑。
